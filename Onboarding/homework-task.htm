    Homework?!?

    Homework?!?

    Jan 31, 20253 min read

    Not really homework… just some tasks to get you started.

    Assuming you have read the Introduction to Benchmarking & Example Benchmarks pages. You should now be familiar with the concept of benchmarking and the framework we will use in this project. To test your understanding, you should try to complete the following “homework” tasks.

    You can submit your work in our Discord Server in the hotpotqa

    Review a Benchmark Paper:

    Your task is to find an LLM benchmark and read the associated paper. Then, summarise the paper using our framework of: task selection, dataset creation, and evaluation metrics. If you would like, you can explore how they conducted their experiments to test models on their created benchmark.

    Benchmark Shortcomings:

    From the benchmarks we have reviewed or any others you have come across, list a few ways in which the chosen benchmark could be improved. For example, the SWE-bench focuses solely on Python code, and as such, an improvement would be to extend to other coding languages (yes I took one of the easy answers so you will have to think harder).

    Testing Benchmark Pipelines:

    A major benefit of reading different benchmark papers is that you can see the entire method that they have employed to go from nothing to a dataset to an evaluation metric. Some benchmarks, like SWE-bench, open-source their code for things like data collection. Testing their pipelines or recreating pipelines from a paper is a good way to get “hands-on”. This will translate to our project since many of the things you will learn can be directly applied. For example, if you write an efficient data collection pipeline to extract information from different GitHub repositories, that could be directly used to create our own benchmark.

    “Why should I do these homework tasks?”

    That is a great question.

    First, reading papers in the field helps you get a good understanding of what is already out there. You can see flaws in other methods or find areas that need to be worked on. For anyone looking to get into paper writing or doing research, this stage is essentially a literature review.

    Speaking of writing papers, you will see that almost all the papers you read will have a “related works” section or that in the introduction there is discussion of the surrounding field. By doing the above tasks, you are essentially writing these sections upfront.

    Finally, while you can work on a project without considering other works, I personally believe contextualising our benchmark is important. Particularly when publishing is one of the goals, providing evidence of the benchmark’s uniqueness and stating its improvements over others is helpful. This allows us to work with a clear idea of what problem we are tackling.
